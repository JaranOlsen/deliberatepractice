# Data Map

This directory contains both hand-edited runtime source files and generated QA artifacts. Do not treat every file here as an equally authoritative editing surface.

## Hand-Edited Runtime Source

- `skills.js`: production and experimental skill metadata shown in the app.
- `cases.js`: public case list, brief case text, current client voice, difficulty, and access tier.
- `caseBibles.js`: editorial case voice/formulation constraints used for content QA.
- `statements.js`: English source of truth for practice statements and suggested responses.
- `translations.js`: UI strings, Norwegian skill/case overrides, and Norwegian statement localizations.
- `glossary.js`: terms surfaced in the app glossary.
- `contentMeta.js`: content revision, production skill order, benchmark mapping, review statuses, and risk/QA taxonomies.
- `contentReviewOverrides.js`: manual metadata overrides for specific content items.
- `index.js`: runtime assembly layer that combines the source files and generated registry.

## Generated Artifacts

These files are generated by `npm run build:content` and are committed because `npm test` validates them:

- `benchmarkBank.js`
- `contentRegistry.js`
- `contentInventory.json`
- `contentQualityReport.json`
- `contentReviewQueue.json`
- `contentSpotCheckPacket.json`
- `blindCaseVoiceReview.json`
- `blindCaseVoiceAnswerKey.json`
- `blindDifficultyReview.json`
- `blindDifficultyAnswerKey.json`
- `contentAcceptanceSummary.json`
- `contentAcceptanceChecklist.md`

Regenerate these after changing `statements.js`, `translations.js`, `cases.js`, `caseBibles.js`, `contentMeta.js`, `contentReviewOverrides.js`, or `src/md/EFT_Exercises_Extracted.md`.

## Removed Snapshot Files

- `statements copy.js`
- `translations copy.js`

These were obsolete accidental snapshots. They were not imported by the app or build scripts and should not be recreated.

## Editing Rule

Use this order for content passes: marker suitability, response skill purity, Norwegian naturalness. Keep stable IDs and the existing data shape unless the app schema is intentionally changed.
