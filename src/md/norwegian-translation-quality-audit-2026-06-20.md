# Norwegian Translation Quality Audit

Date: 2026-06-20

## Purpose

This audit scores the Norwegian Bokmål client statements and suggested therapist responses as Norwegian training content, not as literal parity with the English source. The English content is now stable enough that dedicated Bokmål polish is useful.

The core question is: would this sound natural, clinically precise, and speakable to Norwegian EFT trainees and role-play clients?

## Rating Scale

- 9.0: Natural, idiomatic, clinically precise Bokmål. Reads like it was originally written in Norwegian.
- 8.0-8.9: Strong/app-ready Bokmål. Some phrases still sound translated or slightly repetitive, but the training task is clear and usable.
- 7.0-7.9: Usable but visibly translated. Needs targeted polish before it feels professionally natural.
- 6.0-6.9: Uneven. Meaning is mostly recoverable, but tone, idiom, or clinical wording may interfere with practice.
- Below 6.0: Not acceptable for production training without rewrite.

## Method

- Confirmed all 1080 case-matrix items have Norwegian statement and suggestion translations.
- Scanned by skill and difficulty for repeated response openings and common literal-translation motifs:
  - `la oss`, `kan vi`, `vi kan`
  - `legg merke`, `akkurat nå`, `kroppen`, `gir mening`
  - `bli ved`, `være med`, `lander`, `treffer`, `i deg`, `inni deg`
  - `reparasjon`, `skanning`, `på innsiden`
- Manually spot-checked flagged samples and representative items in each tier.
- Scored client statements and suggested responses separately because the statements are often more natural than the therapist response language.

## Highest-Value Norwegian Polish Targets

1. Marker Recognition and Chair Work, moderate/easy.
   - Main issue: technically correct chair-work setup, but too many responses still sound like translated instructions rather than a Norwegian therapist speaking.
   - Typical motifs: `kan vi`, long multi-step sentence chains, and repeated task-setup wording.

2. Self-Disclosure, hard/easy.
   - Main issue: response content is clinically boundaried, but `la oss`, `vi kan`, and return-to-client phrases feel formulaic.
   - Needs more natural therapist warmth and fewer English-shaped sentence endings.

3. Staying in Contact with Intense Affect, moderate/easy.
   - Main issue: content is clinically careful, but Norwegian often sounds procedural: `vi kan`, repeated `kroppen`, and translated grounding/contact language.
   - Needs warmer, simpler, more spoken presence.

4. Providing Treatment Rationale, easy/moderate/hard.
   - Main issue: rationales are accurate, but several sound written or psychoeducational rather than spoken.
   - Needs fewer `vi kan` starts and more direct everyday Norwegian.

5. Alliance Repair, hard/moderate.
   - Main issue: clinically strong, but often too long and formal for rupture repair in Norwegian.
   - Needs shorter ownership language and more speakable repair questions.

## Implementation Update: Marker Recognition and Chair Work Hard

A hard-tier Norwegian chair-work polish revised all 30 Aisha, David, and Marcus suggested-response localizations.

Current hard chair-work Norwegian state after this pass:

- English source content stayed unchanged.
- Client statement translations stayed unchanged because they were already stronger than the setup responses.
- Response naturalness improved from about 7.2/10 to about 8.0/10.
- Overall tier score improved from about 7.7/10 to about 8.1/10.
- Targeted scan now shows zero `kan vi`, zero `vi kan`, zero `uavsluttet oppgjør`, and zero repeated response starts in the hard tier.
- Chair-work setup is still explicit, but the Norwegian is shorter, less translated, and more speakable.

## Working Rule Going Forward

Since English is now good enough for the next phase, dedicated Norwegian passes should:

1. Keep English unchanged unless a translation problem reveals a real source issue.
2. Work one Norwegian slice at a time, usually 20-40 items.
3. Prioritize response naturalness before client-statement polish, because responses currently carry more translation artifacts.
4. Preserve clinical function over literal parity.
5. Run `npm run build:content`, `npm test`, and `npm run build` after each translation pass.
